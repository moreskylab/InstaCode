Python Collections Quickref

deque

O(1) append and appendleft — use for BFS queues & sliding-window

append(x) → O(1)
appendleft(x) → O(1)
pop() → O(1)
popleft() → O(1)
list.insert(0,x) → O(n)

defaultdict

Auto-initialises missing keys — no KeyError

defaultdict(list)
defaultdict(int)
defaultdict(set)
Use for: adjacency lists, grouping, frequency maps

Counter

Frequency count in O(n)

Counter(iterable) → O(n)
.most_common(k) → top k
c1 + c2 → combine
c1 & c2 → intersect
Use for: anagram, frequency, histogram

heapq

Min-heap; negate values for max-heap

heapify(list) → O(n)
heappush(h, x) → O(log n)
heappop(h) → O(log n)
nlargest/nsmallest → O(n log k)
Use for: Top-K, Dijkstra, median stream

During the Interview

👂

Listen for Constraints

Every constraint is a hint — sorted? → binary search. K elements? → heap. Contiguous? → sliding window.

❓

Clarify Before Coding

Can input be null/empty? Sorted? Mutatable? What's the return type for edge cases?

🗣️

Think Out Loud

Narrate every decision — interviewers evaluate thought process, not just the final answer.

🐢

Brute Force First

Establish correctness first, state its complexity, then identify and eliminate the bottleneck.

🧪

Test Before "Done"

Dry-run: happy path, empty input, single element, all same, max constraints, overflow edge.

📊

State Complexity Proactively

Always close with "Time: O(n log n) · Space: O(n)" — before the interviewer asks.
